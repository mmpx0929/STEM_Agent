# STEM_Agent 配置规范说明

更新时间：2026-03-29

## 1. 单实验配置文件位置

路径：`config/experiments/*.config.js`

示例：
- `config/experiments/science-01.config.js`
- `config/experiments/engineering-02.config.js`

## 2. 单实验配置标准字段

每个实验配置至少包含：

```js
export default {
  id: 'science-01',
  templateType: 'science', // science | engineering
  legacyId: 1,
  title: '实验标题',
  summary: '实验摘要',
  status: 'enabled',

  intro: {
    homeVideoUrl: '/static/Introductory-video/home.mp4',
    step1VideoUrl: '/static/experiments/science-01/videos/intro.mp4'
  },

  voiceGuide: {
    stepGuideTextMap: {
      step1: '语音引导文案',
      step2: '语音引导文案'
    },
    stepSceneMap: {
      step1: 'principle',
      step2: 'planning'
    }
  },

  flowCopy: {
    tips: {
      ruleTitle: '过关小提示',
      missingTitle: '你还差这一步',
      readyTitle: '可以继续啦'
    },
    stepAction: {
      completeAllText: '全部步骤都完成了，点击“完成流程”吧。',
      toNextTemplate: '这一步已经完成，下一步去“{nextTitle}”。'
    },
    stepRuleTips: {
      step1: '步骤提示文案',
      step2: '步骤提示文案'
    },
    stageTitleMap: {
      step1: '阶段标题',
      step2: '阶段标题'
    }
  },

  flowTemplate: { /* 步骤配置模板 */ },

  assets: {
    images: { /* 自动抽取或手动维护 */ },
    videos: { /* 自动抽取或手动维护 */ }
  }
}
```

## 3. 注册规范

### 3.1 实验注册
在 `config/registry/experiment-registry.js` 注册配置：
- 引入对应 `*.config.js`
- 放入 `EXPERIMENT_CONFIGS`
- 同步 `ORDER` 顺序

### 3.2 分类注册
在 `config/registry/category-registry.js` 维护分类：
- `science`
- `engineering`

## 4. 首页目录展示规范

首页由 `config/experiment-catalog.js` 生成：
- 不直接硬编码所有实验
- 统一从分类注册表读取
- 每个实验卡片展示 `id/title/summary/step1VideoUrl`

## 5. 语音引导配置规范

语音引导文案按实验配置维护：
- 字段：`voiceGuide.stepGuideTextMap`
- 场景：`voiceGuide.stepSceneMap`

当前流程页读取策略：
1. 优先读取当前实验 `flowTemplate.voiceGuide`
2. 未配置时使用页面内置 fallback 文案

建议：
- 正式内容只改实验配置文件
- fallback 文案只作为兜底，不作为主编辑入口

## 6. 流程提示文案入口（flowCopy）

流程提示区文案统一维护在每个实验配置的 `flowCopy`：
- 标题区：`flowCopy.tips`
- 过关提示正文：`flowCopy.stepRuleTips`
- 继续提示模板：`flowCopy.stepAction`
- 顶部阶段标题：`flowCopy.stageTitleMap`

页面侧通过 `utils/flowCopyResolver.js` 统一读取：
1. 优先读取 `config/experiments/<id>.config.js` 中 `flowCopy`  
2. 自动兼容 `flowTemplate.ruleTips`（历史字段）  
3. 缺失时回退到 `config/flow-copy-presets.js` 的默认文案  

建议：
- 实验差异文案优先改各实验 `flowCopy`
- 仅跨实验统一口径时改 `flow-copy-presets.js`

## 7. 素材路径规范

- 视频：`/static/experiments/<experiment-id>/videos/...`
- 图片：`/static/experiments/<experiment-id>/images/...`
- 首页导学：`/static/Introductory-video/...`

建议命名：
- 步骤场景视频：`5-1.mp4`、`5-2.mp4`
- 虚拟实验视频：`6-1.mp4`、`6-2.mp4`
- 图纸：`4-1.jpg`、`4-2.jpg`

## 8. 新增实验推荐流程

1. 复制一个最接近的 `*.config.js` 作为模板
2. 修改 `id/title/summary/intro/voiceGuide`
3. 按需补充 `flowCopy`（提示文案、阶段标题）
4. 更新 `flowTemplate`（步骤内容、目标、材料、测试等）
5. 确认 `assets` 覆盖到当前实验素材
6. 在 `experiment-registry.js` 注册
7. 本地验证首页入口、流程页、记录页回显

## 9. 禁止项

- 禁止在页面组件里写死某个实验的素材绝对路径
- 禁止新增“共享大模板”继续堆实验差异
- 禁止绕过 registry 直接在页面构造实验对象
