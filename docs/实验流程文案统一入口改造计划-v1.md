## 1. 目标与原则

### 1.1 改造目标
- 将“每个实验流程界面”的关键提示文案统一收敛到**对应实验配置文件**（`config/experiments/*.config.js`）。
- 后续改文案时，不再反复进入 `pages/experiment/flow.vue`、`pages/engineering/flow.vue` 改逻辑常量。
- 保持现有交互、视觉样式、流程判定规则不变。

### 1.2 约束原则
- 不改业务流程与校验逻辑，只调整文案来源。
- 兼容当前已存在的配置字段，采用“增量接入 + fallback”方式迁移。
- 新字段优先放在每个实验配置中；全局只保留默认兜底。

## 3. 统一入口设计（目标结构）

为每个实验配置新增（或规范化）统一文案块，建议命名：`flowCopy`。

示例结构（草案）：

```js
flowCopy: {
  tips: {
    ruleTitle: '过关小提示',
    missingTitle: '你还差这一步',
    readyTitle: '可以继续啦',
    readyText: '这一步已经完成，点击“继续下一步”吧。'
  },
  stepRuleTips: {
    step1: '...',
    step2: '...',
    // ...
    step10: '...' // 工程到 step9
  },
  stepAction: {
    completeAllText: '全部步骤都完成了，点击“完成流程”吧。',
    toNextTemplate: '这一步已经完成，下一步去“{nextTitle}”。'
  },
  stageTitleMap: {
    step1: '场景问题导入',
    step2: '工程方案设计',
    // ...
  }
}
```

说明：
- `flowCopy` 放在每个实验配置根对象，避免散落在 `flowTemplate` 与页面常量之间。
- `stageTitleMap` 主要服务“导航栏标题/阶段提示”统一来源，避免页面硬编码。
- 与现有 `voiceGuide` 并行，不互相覆盖。

## 4. 代码改造计划（分阶段）

## 4.1 Phase A：补齐解析层（先不动页面行为）
- 新增文案解析工具（建议文件）：`utils/flowCopyResolver.js`。
- 输入：`experimentId`、`templateType`、`stepKey`、`nextTitle`。
- 输出：页面所需文案（标题、规则提示、下一步提示、完成提示）。
- 兜底顺序（建议）：
  1. 当前实验 `flowCopy.*`
  2. 全局默认 `config/kid-copy.js` / 页面默认值
  3. 最终硬编码兜底（防空字符串）

## 4.2 Phase B：页面接入统一入口
- 科学流程页：`pages/experiment/flow.vue`
  - `ruleTitle`、`getRuleTip`、`blockerTitle`、`stepActionHint` 改为调用 resolver。
- 工程流程页：`pages/engineering/flow.vue`
  - 同上；并把当前步骤标题映射也改为配置优先。

## 4.3 Phase C：配置迁移（每个实验）
- 科学 3 个实验：补全 `flowCopy.stepRuleTips(step1~step10)`。
- 工程 8 个实验：补全 `flowCopy.stepRuleTips(step1~step9)` + `stageTitleMap`。
- 迁移完成后，仅保留页面层默认值用于兜底，不作为主入口。

## 4.4 Phase D：校验与回归
- 逐实验校验：
  - 黄卡标题/正文是否跟配置一致。
  - 绿卡标题/正文是否跟配置一致。
  - “下一步去 xxx”中 `xxx` 是否正确映射。
  - 导航标题是否跟步骤一致（含工程“工程方案设计”聚合阶段）。
- 增加快速检查脚本（可选）：
  - 扫描每个实验配置的 `flowCopy` 完整性。

## 6. 验收标准

- 任一实验流程页的提示文案都能在其对应 `config/experiments/<id>.config.js` 中找到并修改。
- 页面层不再有大段实验特定提示文案（仅保留通用兜底）。
- 修改单个实验文案不会影响其他实验。
- 不影响现有流程校验、按钮禁用、跳步限制和样式。

## 7. 风险与回滚

## 7.1 风险
- 风险1：字段迁移不完整导致某步骤提示为空。
- 风险2：科学/工程步骤数量不同（10 vs 9）导致索引映射错误。
- 风险3：历史编码格式不一致引发文案乱码（个别旧文件存在编码遗留）。

## 7.2 回滚策略
- 保留页面默认文案作为兜底，逐步切换开关。
- 采用“先接入 resolver + 不删除旧文案”方式发布，回归稳定后再清理冗余常量。

## 8. 执行顺序建议

1. 先完成 `flowCopy` 字段规范与 resolver（最小可运行）。  
2. 先接工程流程页，再接科学流程页（或并行）。  
3. 批量迁移 11 个实验配置文案。  
4. 回归校验后，再做页面内旧常量清理。
