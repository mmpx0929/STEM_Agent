# STEM_Agent 项目结构说明

更新时间：2026-03-29

## 1. 项目目标
`STEM_Agent` 是从原项目迁移出的新框架，目标是：
- 保留首页、科学流程、工程流程、记录、成长、数据记录等核心功能
- 将实验配置按“每个实验一个文件”拆分管理
- 清理历史冗余目录和重复副本，降低维护成本

## 3. 关键配置入口

### 3.1 实验注册表
- 文件：`config/registry/experiment-registry.js`
- 作用：统一注册所有实验配置（按实验 ID 查找）

### 3.2 分类注册表
- 文件：`config/registry/category-registry.js`
- 作用：定义科学类 / 工程类分类及首页展示结构

### 3.3 首页目录配置
- 文件：`config/experiment-catalog.js`
- 作用：首页分类、标题、副标题、导学视频入口

### 3.4 流程模板读取
- 文件：`config/science-flow-templates.js`
- 文件：`config/engineering-flow-templates.js`
- 作用：兼容原有组件调用入口，底层从新 registry 读取对应实验配置

## 4. 页面与配置关系

- `pages/index/index.vue`
  - 读取 `experiment-catalog.js` 展示实验目录与导学入口
- `pages/experiment/flow.vue`
  - 读取科学实验模板与步骤配置
  - AI 引导文案/场景优先读取实验配置 `voiceGuide`
- `pages/engineering/flow.vue`
  - 读取工程实验模板与步骤配置
  - AI 引导文案/场景优先读取实验配置 `voiceGuide`
- `record/growth/data-record`
  - 通过实验 ID 与模板类型读取配置并完成展示/回显

## 5. 素材管理规则

- 每个实验素材统一放在 `static/experiments/<id>/images|videos`
- 实验配置文件中维护对应素材路径（图片/视频）
- 页面组件不再硬编码具体实验素材路径

## 6. 已完成的冗余清理

仅在 `STEM_Agent` 内完成清理，未改动原项目目录：
- 删除重复副本目录：
  - `config/config`
  - `utils/utils`
  - `pages/*/*` 的重复嵌套副本
  - `components/*/*` 的重复嵌套副本
  - `static/science-01/science-01`
- 统一实验素材目录：
  - `static/science-01..03` → `static/experiments/science-01..03`
  - `static/engineering-01..08` → `static/experiments/engineering-01..08`
- 删除无引用旧配置：
  - `config/experiments.js`
  - `config/lab-config.js`

## 7. 维护建议

- 新增实验时，优先新增 `config/experiments/<id>.config.js`
- 再在 `experiment-registry.js` 注册
- 每个实验直接维护独立配置，不再依赖 legacy 模板
- 页面层如需新文案，优先写入对应实验配置 `voiceGuide`
